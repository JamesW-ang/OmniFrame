## 日常维护

### 每日检查清单

| 时间点 | 检查项目 | 方法 |
|:---|:---|:---|
| 开机前 | 检查硬件连接 | 目视检查网线、串口线 |
| 开机后 | 检查应用状态 | 查看登录界面是否正常 |
| 运行中 | 监控运行日志 | 检查是否有异常告警 |
| 关机前 | 数据备份 | 确认当日数据已保存 |

### 日常操作规范

1. **启动顺序**
   ```
   开机 → 等待系统就绪 → 启动应用 → 登录 → 检查状态 → 开始生产
   ```

2. **关机顺序**
   ```
   停止生产 → 保存数据 → 退出应用 → 关机
   ```

3. **紧急停止**
   - 立即按下急停按钮
   - 切断设备电源（如需要）
   - 记录故障现象

## 故障排除

### 故障分类

| 等级 | 定义 | 处理时限 |
|:---|:---|:---|
| P0 | 系统无法启动 | 2小时 |
| P1 | 功能严重异常 | 4小时 |
| P2 | 功能轻度异常 | 24小时 |
| P3 | 优化改进 | 72小时 |

### 运动控制类

#### 故障：轴无法移动

**排查步骤**:

1. 检查轴是否已使能
   - 查看使能状态指示灯
   - 如未使能，执行使能操作

2. 检查限位触发
   - 查看限位状态
   - 如触发，手动移开或复位

3. 检查运动参数
   - 速度设置是否合理
   - 加速度是否过大

4. 检查硬件连接
   - 确认控制卡连接正常
   - 尝试重启控制卡

**应急处理**:
```powershell
# 紧急停止所有轴
# 按下急停按钮
# 切断驱动电源
```

#### 故障：运动抖动

**可能原因**:
- 速度/加速度设置过大
- 机械卡顿
- 驱动器参数不匹配

**处理方法**:
1. 降低速度和加速度
2. 检查机械润滑
3. 重新整定驱动器参数

### PLC 通信类

#### 故障：PLC 连接断开

**排查步骤**:

1. 检查网络连接
   ```powershell
   ping 192.168.1.101
   ```

2. 检查 PLC 状态
   - 查看 PLC RUN/STOP 指示灯
   - 检查 PLC 错误代码

3. 检查配置文件
   - IP 地址是否正确
   - 端口是否正确

4. 重启 PLC 通信
   - 断开重连

#### 故障：寄存器读写异常

**排查步骤**:

1. 确认寄存器地址
   - 检查配置文件中的地址
   - 确认 PLC 侧地址映射

2. 检查数据类型
   - 确认读写数据类型一致
   - 检查字节序问题

3. 验证读写权限
   - 确认 PLC 侧允许读写

### 数据管理类

#### 故障：数据不写入

**排查步骤**:

1. 检查数据库连接
   - 查看连接字符串
   - 测试连接

2. 检查磁盘空间
   - SQLite 文件是否过大
   - 是否有足够空间

3. 检查权限
   - 应用是否有写入权限

**处理方法**:
```sql
-- 清理历史数据（谨慎操作）
DELETE FROM ProductData WHERE CreateTime < DATEADD(month, -3, GETDATE());
```

### 界面显示类

#### 故障：界面卡顿

**排查步骤**:

1. 检查资源占用
   - CPU 使用率
   - 内存使用率

2. 检查显示刷新
   - 是否有大量数据刷新
   - 是否有死循环

3. 检查网络
   - 远程监控是否占用带宽

**处理方法**:
- 重启应用程序
- 如频繁发生，联系技术支持

## 日志分析

### 日志文件位置

```
OmniFrame/
└── Logs/
    ├── log-20260510.txt      # 应用日志
    ├── audit-20260510.json   # 审计日志
    └── alarm-20260510.txt    # 告警日志
```

### 日志级别

| 级别 | 标识 | 说明 |
|:---|:---|:---|
| INFO | 信息 | 正常运行信息 |
| WARNING | 警告 | 需要注意但不影响运行 |
| ERROR | 错误 | 功能异常，需要处理 |
| FATAL | 致命 | 系统无法继续运行 |

### 关键日志关键词

| 关键词 | 含义 | 处理建议 |
|:---|:---|:---|
| `Connection refused` | 连接被拒绝 | 检查目标服务是否启动 |
| `Timeout` | 超时 | 检查网络或目标响应 |
| `Exception` | 异常 | 查看完整堆栈跟踪 |
| `Alarm` | 告警 | 查看告警详情并处理 |
| `Config changed` | 配置变更 | 确认是否为预期变更 |

### 日志分析方法

1. **按时间查找**
   ```
   发现问题时间: 14:30
   在日志中搜索该时间点前后的记录
   ```

2. **按关键字查找**
   ```
   搜索关键词: ERROR
   查找所有错误记录
   ```

3. **按模块查找**
   ```
   搜索关键词: [Motion]
   查找运动控制相关日志
   ```

## 应急处理

### 系统无法启动

**应急步骤**:

1. 检查错误日志
   - 查看 `Logs/log-*.txt`
   - 记录错误信息

2. 常见解决方案:
   - 配置文件损坏 → 恢复备份
   - 端口被占用 → 修改端口或结束占用进程
   - 数据库损坏 → 恢复数据库备份

3. 联系技术支持:
   - 提供错误日志
   - 提供系统环境信息

### 数据损坏

**应急步骤**:

1. 立即停止写入
   - 防止数据进一步损坏

2. 评估损坏范围
   - 检查哪些数据受影响

3. 恢复备份
   - 使用最近的完整备份恢复

4. 补录数据（如需要）
   - 根据生产记录补录

### 网络攻击

**应急步骤**:

1. 立即断开网络
   - 隔离受攻击设备

2. 评估影响范围
   - 检查是否有数据泄露

3. 报告事件
   - 联系信息安全部门

4. 恢复系统
   - 在安全环境下重建系统
