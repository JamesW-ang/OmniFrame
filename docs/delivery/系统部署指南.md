## 概述

本指南适用于 OmniFrame 3.0.0 版本的正式生产环境部署。

### 适用范围

- 新系统首次部署
- 系统升级部署
- 故障恢复部署

### 部署模式

| 模式 | 说明 | 适用场景 |
|:---|:---|:---|
| 单机模式 | 所有组件部署在同一台机器 | 小型生产线 |
| 分布模式 | 各组件分离部署 | 大型生产线 |

## 部署架构

### 单机模式架构

```
┌─────────────────────────────────────────────────┐
│                  工控机 (Host)                    │
│  ┌─────────────────────────────────────────┐   │
│  │           OmniFrame 主程序               │   │
│  ├─────────────────────────────────────────┤   │
│  │  MotionIO  │  PLC  │  RemoteMonitor     │   │
│  ├─────────────────────────────────────────┤   │
│  │           SQLite Database               │   │
│  └─────────────────────────────────────────┘   │
│         ↑              ↑              ↑         │
│    运动控制卡      PLC设备      网络/扫码枪      │
└─────────────────────────────────────────────────┘
```

### 网络要求

| 项目 | 要求 |
|:---|:---|
| 工控机 IP | 固定 IP，建议 192.168.1.x 网段 |
| MES 连接 | 支持 HTTP/HTTPS |
| 远程监控端口 | WebSocket: 8080, REST API: 8081 |

## 环境准备

### 硬件要求

| 项目 | 最低配置 | 推荐配置 |
|:---|:---|:---|

| 项目 | 版本要求 |
|:---|:---|

| PLC 驱动 | 厂家提供 |

需要开放以下端口：

```
端口    协议    用途
--------------------------

8081    TCP     REST API
1433    TCP     SQL Server (如使用)
22      TCP     SSH (远程维护)
```

## 安装步骤

### 步骤 1：系统准备

1. 安装 Windows 10/11 Pro
2. 配置固定 IP 地址
3. 加入域（可选）
4. 创建专用用户账户

### 步骤 2：安装依赖

```powershell
# 安装 .NET Framework 4.8
# 下载地址：https://dotnet.microsoft.com/download/dotnet-framework/net48

# 安装 Visual C++ Redistributable
# 下载地址：https://aka.ms/vs/17/release/vc_redist.x64.exe
```

### 步骤 3：安装 OmniFrame

1. 复制安装文件到工控机
2. 运行安装向导
3. 选择安装路径（建议：D:\OmniFrame）
4. 创建桌面快捷方式

### 步骤 4：配置硬件连接

```
运动控制卡:
  - 安装 PCI/PCIe 卡槽
  - 安装厂家驱动
  - 确认设备在设备管理器中可见

PLC:
  - 配置网络参数 (IP: 192.168.1.100, Port: 502)
  - 确认网络连通性

扫码枪:
  - 配置 RS232 或 TCP 连接
  - 测试通信
```

### 步骤 5：配置文件编辑

修改 `Config/` 目录下的配置文件：

```xml
<!-- MotionConfig.xml -->
<MotionConfig>
  <Brand>PCIeM60</Brand>
  <IP>192.168.1.100</IP>
  <AxisCount>4</AxisCount>
</MotionConfig>

<!-- PlcConfig.xml -->
<PlcConfig>
  <Brand>ModbusTCP</Brand>
  <IP>192.168.1.101</IP>
  <Port>502</Port>
</PlcConfig>
```

## 配置说明

### 环境变量

| 变量名 | 必填 | 说明 |
|:---|:---|:---|
| `OMNIFRAME_CONFIG_ENCRYPTION_KEY` | 是 | MES 配置加密密钥 |
| `OMNIFRAME_MES_AES_KEY` | 是 | MES 通信 AES 密钥（32字符） |
| `OMNIFRAME_BARCODE_HOST` | 是 | 扫码枪 IP 地址 |
| `OMNIFRAME_BARCODE_PORT` | 是 | 扫码枪端口（默认 5000） |
| `OMNIFRAME_SIMULATION` | 否 | `=1` 启用仿真模式（跳过硬件检测） |

> 注意：v3.1.0 起，`CONFIG_ENCRYPTION_KEY`、`MES_AES_KEY`、`BARCODE_HOST`、`BARCODE_PORT` 均在生产模式下为**必填**，缺失时程序拒绝启动。

### 配置文件

| 文件名 | 说明 |
|:---|:---|
| `MotionConfig.xml` | 运动控制参数 |
| `PlcConfig.xml` | PLC 通信参数 |
| `IoConfig.xml` | IO 点位配置 |
| `SystemConfig.xml` | 系统参数 |
| `BlockCut.xml` | BlockCut 工站配置 |
| `appsettings.json` | **v3.1.0+** 产线运行参数（重连/看门狗/系统） |

### 日志配置

日志文件位置：`Logs/`

| 日志类型 | 文件名模式 |
|:---|:---|
| 应用日志 | `log-YYYYMMDD.txt` |
| 审计日志 | `audit-YYYYMMDD.json` |
| 告警日志 | `alarm-YYYYMMDD.txt` |
| 看门狗诊断 | `Watchdog/Watchdog_*.log` |
| 崩溃转储 | `CrashDumps/crash_*.log` |

### JSON 统一配置（v3.1.0+）

`Config/appsettings.json` 在首次运行时自动创建，不建议手动修改。

```json
{
  "Reconnect": {
    "Enabled": true,
    "MaxBackoffMs": 15000,
    "AlarmThreshold": 3,
    "MaxRetries": 10
  },
  "Watchdog": {
    "Enabled": true,
    "IntervalMs": 1000,
    "FailureThreshold": 5
  },
  "System": {
    "LogLevel": "Info",
    "EnableWatchdog": true,
    "EnableAutoReconnect": true
  }
}
```

### v3.1.0 新增服务说明

| 服务 | 启停控制 | 文件 |
|:---|:---|:---|
| **ReconnectionService** | `System.EnableAutoReconnect` | `OmniFrame.Core/ReconnectionService.cs` |
| **WatchdogService** | `System.EnableWatchdog` | `OmniFrame.Core/WatchdogService.cs` |
| 崩溃转储 | 始终启用 | `Program.cs` |

所有服务在 `Program.cs` 的 `StartupInfrastructure()` 中自动启动，在 `ShutdownInfrastructure()` 中自动停止，无需手动干预。关闭产线功能只需修改 `appsettings.json` 中对应开关为 `false`。

## 常见问题

### Q1: 运动控制卡无法识别

**A**: 
1. 检查 PCI/PCIe 插槽是否插紧
2. 在设备管理器中查看是否有未知设备
3. 重新安装驱动程序
4. 尝试其他插槽

### Q2: PLC 连接失败

**A**:
1. 确认网络线连接正常
2. 使用 ping 命令测试连通性
3. 检查 PLC IP 地址和端口配置
4. 确认 PLC 侧允许 Modbus TCP 连接

### Q3: 数据库连接异常

**A**:
1. 检查 SQL Server 服务是否启动
2. 确认连接字符串配置正确
3. 检查防火墙是否阻止 1433 端口
4. 查看日志中的详细错误信息

### Q4: 远程监控无法访问

**A**:
1. 确认 WebSocket 端口 8080 未被占用
2. 检查防火墙规则
3. 确认客户端 IP 在允许列表中

## 回滚方案

如部署失败，按以下步骤回滚：

1. 停止 OmniFrame 服务
2. 恢复配置文件备份
3. 如有必要，恢复数据库备份
4. 重启应用验证
