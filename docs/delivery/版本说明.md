## 版本历史

### v3.1.0 (2026-05-10)

**类型**: 产线就绪版本

#### 新增特性

| 特性 | 组件 | 说明 |
|:---|:---|:---|
| 设备自动重连 | `ReconnectionService` | 断连自动恢复，指数退避重试，超过阈值触发报警 |
| 系统看门狗 | `WatchdogService` | 1s 心跳检查，连续5次失败触发安全停机+诊断快照 |
| 统一配置 | `JsonConfigProvider`（规划中） | appsettings.json 统一入口，覆盖重连/看门狗/系统参数 |
| 工站自动恢复 | `BlockCutStationBase` | RunWithAutoRecoveryAsync 包装器，异常自动重试 |
| 崩溃转储 | `Program.cs` | 未处理异常自动写入 Logs/CrashDumps/ |

#### 新增文件

| 文件 | 说明 |
|:---|:---|
| `src/OmniFrame.Core/ReconnectionService.cs` | 设备自动重连服务 |
| `src/OmniFrame.Core/WatchdogService.cs` | 系统看门狗服务 |
| `src/OmniFrame.Core/JsonConfigProvider.cs`（规划中） | JSON 统一配置提供器 |
| `Config/appsettings.json` | 产线运行配置文件（首次运行自动创建） |

#### 关闭产线功能

在 `Config/appsettings.json` 中设置以下值可恢复到 v3.0.x 行为：

```json
{
  "System": {
    "EnableWatchdog": false,
    "EnableAutoReconnect": false
  }
}
```

### v3.0.1 (2026-05-10)

**类型**: Bug 修复版本

#### 修复内容

| 序号 | 问题编号 | 描述 | 影响模块 |
|:---|:---|:---|:---|
| 1 | OMNI-001 | 插件管理界面 SplitterDistance 错误导致界面打不开 | PluginManagerForm |
| 2 | OMNI-002 | 操作日志查询无结果显示 | OperationLogForm |
| 3 | OMNI-003 | BlockCut 配置不显示内容 | ConfigForm |
| 4 | OMNI-004 | 环境变量缺失导致启动失败 | Program.cs |
| 5 | OMNI-005 | BlockCutConfig 仿真模式默认值为 false | BlockCutConfig |

#### 修复详情

**1. 插件管理 SplitterDistance 错误**
- **问题**: `SplitContainer` 在创建时设置 `Panel1MinSize=350` 和 `Panel2MinSize=350`，但容器宽度可能为0，导致 `SplitterDistance` 计算错误
- **修复**: 将尺寸设置移到 `HandleCreated` 事件中，动态计算
- **文件**: `PluginManagerForm.cs`

**2. 操作日志不显示**
- **问题1**: 日期范围默认7天，但历史数据为1个月前
- **问题2**: `UpdateUserList()` 在 `LoadLogs()` 之后调用导致空引用
- **修复**: 调整调用顺序，扩大默认日期范围为1个月
- **文件**: `OperationLogForm.cs`

**3. BlockCut 配置不显示**
- **问题**: `BuildBlockCutTab()` 在 `LoadConfig()` 之后调用
- **修复**: 调整初始化顺序
- **文件**: `ConfigForm.cs`

**4. 环境变量缺失**
- **问题**: `OMNIFRAME_MES_AES_KEY` 和 `OMNIFRAME_BARCODE_HOST` 未设置时应用无法启动
- **修复**: 仿真模式下使用默认值
- **文件**: `Program.cs`

## 版本命名规则

```
v主版本.次版本.修订版本

示例: v3.0.1
- 主版本: 3 (重大架构更新)
- 次版本: 0 (新增功能)
- 修订版本: 1 (Bug修复)
```

## 版本类型

| 类型 | 标识 | 说明 |
|:---|:---|:---|
| Major | 主版本号变更 | 架构调整，不兼容升级 |
| Minor | 次版本号变更 | 新增功能，向后兼容 |
| Patch | 修订号变更 | Bug修复，完全兼容 |

## 升级指南

### 从 v2.x 升级到 v3.0

**注意事项**:
1. v3.0 数据库结构有变更，需要执行数据库迁移脚本
2. 部分 API 有调整，请参考 API 变更文档
3. 配置文件格式有更新，建议使用默认配置生成

**升级步骤**:
1. 备份 v2.x 数据库和配置文件
2. 安装 v3.0
3. 执行数据库迁移工具
4. 验证原有数据完整性

### 从 v3.0.0 升级到 v3.0.1

**注意事项**:
- v3.0.1 为完全兼容的 Bug 修复版本
- 可直接覆盖安装

**升级步骤**:
1. 备份配置文件
2. 停止当前运行的 v3.0.0
3. 安装 v3.0.1
4. 启动应用并验证
